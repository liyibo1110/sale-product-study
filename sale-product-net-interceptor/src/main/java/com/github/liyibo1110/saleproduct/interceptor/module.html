1、流量劫持服务，负责接收调用方的HTTP请求，根据Nacos配置的规则来决定服务是转给php旧服务，还是Java新服务。 2、如果转给php旧服务，则直接透传，如果转给Java新服务，则要做参数转换，再转发。 3、收到响应后返回给调用方，如果是来自php旧服务的响应，则直接透传，如果是来自Java新服务的响应，也要将格式转回php接口原来的响应格式，再返回给php调用方。 4、如果目标是php旧服务，且当前接口开启了流量采集，会对请求参数做一次转换，生成Java服务的格式入参，然后把转换后的Java格式入参，以及php接口的响应一起异步发送给数据质检服务。 5、上述质检服务，只在最初阶段开启（即Java接口上线，但全部流量还只转发给php旧接口），质检是用来提前测试往Java新接口转发的正确性，如果一旦真分流给了Java接口，这个检查就没有意义了。
